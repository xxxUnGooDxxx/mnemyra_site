# SEO: что добавить при публикации

Техническая оптимизация страницы уже выполнена. Для завершения настройки нужен реальный HTTPS-домен.

1. В `index.html` добавить абсолютные адреса:
   - `<link rel="canonical" href="https://ДОМЕН/">`
   - `<meta property="og:url" content="https://ДОМЕН/">`
   - заменить `og-image.png` в `og:image` и `twitter:image` на `https://ДОМЕН/og-image.png`.
2. В JSON-LD добавить `url`, а после публикации приложения — фактический `downloadUrl`.
3. Создать `sitemap.xml` с адресом `https://ДОМЕН/` и добавить `Sitemap: https://ДОМЕН/sitemap.xml` в `robots.txt`.
4. Настроить единственную основную версию домена: HTTPS, с `www` или без `www`; остальные варианты перенаправить на неё постоянным редиректом.
5. Добавить сайт в Google Search Console и Яндекс Вебмастер, подтвердить права, отправить Sitemap и запросить переобход главной страницы.
6. Проверить, что сервер отдаёт `200` для главной, `404` для отсутствующих страниц и корректные MIME-типы для SVG, PNG, CSS, JS и manifest.
7. Проверить, что все кнопки скачивания ведут на актуальную страницу приложения в RuStore.

Не добавляйте `meta keywords`: Google его не использует. Не добавляйте рейтинг, цену или отзывы в структурированные данные, пока они не опубликованы и не видны пользователю на странице.
